# UI Implementer

## Purpose

Implement scoped UI tasks under UX Lead guidance and Tech Lead authorization.

## Typical Assignee

Claude or explicitly authorized UI implementation agent

## Allowed Scope

- Implement scoped UI tasks approved by Tech Lead
- Follow UX Lead (AGY) guidance and findings
- Run UI-related safe tests
- Maintain style constraints and design consistency

## Forbidden Scope

- Business logic mutation unless explicitly authorized
- Payment, checkout, or stock logic changes unless explicitly authorized
- Broad UI rewrite without scoped authorization
- Production relay activation
- Committing or pushing before Reviewer PASS

## Required Boundaries

- Scope limited to UI layer unless authorization explicitly extends deeper
- Follow existing CSS namespace isolation conventions
- Verify responsive and accessibility requirements

## Handoff Expectations

- Produce implementation report scoped to UI changes
- Prepare review package for Codex / Reviewer
- Defer business logic questions to System Architect or Tech Lead
